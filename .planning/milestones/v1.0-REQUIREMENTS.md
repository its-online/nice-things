# Requirements: Nice Things

**Defined:** 2026-03-31
**Core Value:** Saved links must be easy to sync from notes and fast to find again through simple browsing and filtering.

## v1 Requirements

### Sync Pipeline

- [ ] **SYNC-01**: Maintainer can sync bookmark records from `../notes/resources/bookmarks/marks/` into the repo with one command
- [ ] **SYNC-02**: Sync excludes `articles/**` and ignores non-markdown files
- [ ] **SYNC-03**: Sync parses YAML frontmatter from bookmark markdown files without depending on markdown body content
- [ ] **SYNC-04**: Sync skips records missing `bookmark_title` or `bookmark_url` and logs warnings
- [ ] **SYNC-05**: Sync skips records with invalid frontmatter and logs warnings instead of failing the whole build
- [ ] **SYNC-06**: Sync normalizes tags by trimming and lowercasing, with empty or missing tags becoming `[]`
- [ ] **SYNC-07**: Sync normalizes blank or missing `bookmark_done` values to `done: false`
- [ ] **SYNC-08**: Sync derives `categoryPath` from folder segments under `marks`
- [ ] **SYNC-09**: Sync assigns a stable bookmark route key from the source file name and fails with a clear error on duplicates
- [ ] **SYNC-10**: Sync writes normalized bookmark data into a committed generated artifact inside this repo so production builds do not depend on the notes directory

### Archive Browsing

- [ ] **ARCH-01**: Visitor can open a public homepage that lists all synced bookmarks
- [ ] **ARCH-02**: Visitor can search bookmarks on the homepage by title, URL, description, and tags
- [ ] **ARCH-03**: Visitor can filter bookmarks on the homepage by tag
- [ ] **ARCH-04**: Visitor can filter bookmarks on the homepage by category
- [ ] **ARCH-05**: Visitor sees a clear empty state when search or filters return no bookmarks
- [ ] **ARCH-06**: Visitor sees bookmarks in a deterministic default order

### Taxonomy Pages

- [ ] **TAX-01**: Visitor can open a dedicated page for every normalized tag
- [ ] **TAX-02**: Visitor can open a dedicated page for every category path present in the synced bookmarks
- [ ] **TAX-03**: Tag and category pages list only bookmarks that belong to the selected taxonomy value

### Bookmark Detail

- [ ] **DET-01**: Visitor can open a dedicated `/b/[route-key]` page for every synced bookmark
- [ ] **DET-02**: Bookmark detail pages show title, outbound URL, description when present, tags, category path, and source-relative path
- [ ] **DET-03**: Bookmark detail pages include a clear outbound link to the original resource

### Build and Deployment

- [ ] **BLD-01**: `bun run build` runs bookmark sync before `astro build`
- [ ] **BLD-02**: The site builds successfully from committed repo contents without access to the notes directory
- [ ] **BLD-03**: Cloudflare Pages can deploy the site from `dist`
- [ ] **BLD-04**: Build configuration pins the intended Bun version for reproducible deploys
- [ ] **BLD-05**: The site generates all required static routes for bookmarks, tags, and categories without deploy-time 404 gaps

### Verification

- [ ] **TEST-01**: Automated tests verify sync normalization for empty tags, blank `bookmark_done`, invalid frontmatter, and missing required fields
- [ ] **TEST-02**: Automated checks verify duplicate route-key detection aborts sync clearly
- [ ] **TEST-03**: Automated checks verify homepage, tag pages, category pages, and bookmark detail pages build and resolve successfully

## v2 Requirements

### Archive Enhancements

- **NEXT-01**: Visitor can browse related bookmarks from a bookmark detail page
- **NEXT-02**: Visitor can filter bookmarks by source domain
- **NEXT-03**: Visitor can subscribe to bookmark updates via RSS or JSON feeds
- **NEXT-04**: Maintainer can feature or pin selected bookmarks on the homepage

### Future Archive Depth

- **FUT-01**: Visitor can search archived page content beyond bookmark metadata
- **FUT-02**: Visitor can view previews or screenshots for bookmarks
- **FUT-03**: Archive preserves copies of linked pages to reduce link rot

## Out of Scope

| Feature | Reason |
|---------|--------|
| Server-side rendering | v1 is intentionally static for simpler hosting and maintenance |
| Rendering bookmark markdown bodies | v1 is metadata-only to keep import and page generation simple |
| User accounts, likes, comments, or collaboration | This is a public read-only archive, not an application with user state |
| Browser capture, web clipping, or note editing workflows | The external notes repo already owns authoring and capture |
| AI tagging, summarization, or chat features | Not needed to prove retrieval value and would add complexity early |
| Full-text page indexing | Deferred until metadata-only retrieval is no longer enough |
